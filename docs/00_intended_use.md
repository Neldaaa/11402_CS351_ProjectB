# Intended Use: CSV Mini Database & Query Engine

## Overview
Project B implements a lightweight CSV data management system with query capabilities, designed to provide basic database-like functionality for CSV files without requiring a full-fledged database system.

## Primary Use Cases

### 1. Data Management
- Load CSV files into memory for fast access and manipulation
- Store and organize tabular data in a structured format
- Support basic CRUD operations (Create, Read, Update, Delete) on CSV records

### 2. Query Processing
- Execute SQL-like queries on CSV data
- Filter records based on specified conditions
- Project specific columns from datasets
- Sort and aggregate data

### 3. Educational Purpose
- Learn database fundamentals including query parsing and execution
- Understand data structure design and implementation
- Explore in-memory data processing techniques

## Target Users
- Students learning database concepts
- Developers needing simple CSV data querying without database overhead
- Users working with small to medium-sized CSV datasets

## Scope & Limitations
- Optimized for in-memory processing of reasonably-sized datasets
- Intended for learning and prototyping, not production-scale applications
- Focus on core query functionality rather than advanced database features
